Solution overview — the key decisions

One paragraph of orientation before the slides: we turn the firm's mixed documents into structured, auditable data with a cheap-first, escalate-when-unsure design — deterministic rules and a fast model handle the easy majority, a premium model handles the hard tail, and a human handles the rest. The decisions that shape everything:

DecisionIn one line
Classify then routeRules scorer over text → escalate to an LLM if unsure → each doc type gets its own extractor + schema
Extract with DSPyA typed signature = the LLM interface and the validation contract in one object
Model tieringGemini Flash by default, escalate to Claude Opus only on low confidence (speed + cost)
Fail gracefullyNothing guessed — escalate, then a human; every decision recorded
Native auditAppend-only log + Delta history + Unity Catalog lineage/access + MLflow — configured, not built
Async APIJob id returned instantly, processed on a worker, polled for the result
Eval-drivenTest on a gold set before deploy; escalated + corrected cases improve it on the firm's own docs

Cost of all this: ~$0.005/document vs ~$15–25 done by hand — see Cost & ROI.

Talk-track — Slide A
The problem is turning the firm's messy Claims paperwork into structured, queryable, auditable data. Our approach is a cheap-first cascade: a deterministic step handles the easy majority and we only spend an LLM when the system isn't confident — that's what keeps it fast and affordable. The three things the brief asks about are on the slide: failures are escalated, never guessed (worst case a human reviews); it scales by decoupling ingest from extraction and moving to provisioned throughput as volume grows; and the audit trail is native to Databricks — we configure Delta history, Unity Catalog lineage, and MLflow rather than build a logging system. That last point is the main reason we chose Databricks.

Talk-track — Slide B
Follow the green "yes" line — that's the easy ~90%, handled fast and cheap. At each checkpoint, if the system isn't confident it escalates to a stronger model instead of guessing, and a genuine failure goes to a person. The brief's three required failure modes are all here: unparseable output re-asks then escalates, image-only scans go to the vision model, and flaky API calls retry with back-off. Validated data lands in Delta and powers reporting, search, and the future Q&A. And because every escalation and human correction is captured, the system improves on the firm's actual documents over time — a POC that's a hypothesis, not a one-off.
