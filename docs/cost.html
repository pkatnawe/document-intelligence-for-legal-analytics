Cost & ROI

At an automated cost of well under a cent per document against $15–25 by hand, model tokens are a rounding error. The real cost lever is how often a human has to step in — which the evaluation loop drives down over time.

TL;DR
  • ~$0.005 per document blended, versus $15–25 handled by hand — three to four orders of magnitude cheaper.
  • Tiering cuts model cost ~80% versus running the premium model on everything.
  • Token cost is negligible; the real variable cost is the human-review rate, which evaluation reduces over time.

The method

Cost per document = (input tokens × input rate) + (output tokens × output rate), blended across the two model tiers. A typical invoice is ~2,000 input tokens (the text) and ~600 output tokens (the structured JSON). Public per-million-token rates (June 2026) — Databricks bills in DBUs that track these token economics:

TierModelInput / 1MOutput / 1M
1 — defaultGemini Flash$0.50$3.00
2 — escalationClaude Opus 4.8$5.00$25.00

Classification adds ~nothing: the type classifier (rules on text, a lightweight image model on scans) is cheap/local, and only the small low-confidence tail hits an LLM/VLM. Embeddings for future search are a fraction of a cent per document.

Note: these rates are the production tiers (Gemini / Claude). On Free Edition those are disabled, so the POC runs open-weight models that are included — POC token cost is effectively $0. The figures below are the paid-workspace economics the design targets.

The ROI is decisive. Automated extraction costs ~$0.005/doc against ~$15–25 done by hand — three to four orders of magnitude cheaper. At that point the token bill is a rounding error, and the real question becomes the one below.

The real cost driver: human review, not tokens

At scale the variable cost isn't the model — it's the escalation / human-review rate. If even 5% of documents need a minute of a paralegal's time (~$0.50 each), that's ~$0.025/doc — larger than the entire model cost. So the cost story and the quality story are the same story: drive the human-review rate down, which is precisely what the evaluation loop does as the system learns the firm's documents. Token spend is fixed and tiny; human time is the lever.

What changes at high volume

Pay-per-token has per-minute rate limits and isn't built for sustained high throughput; above ~steady production load we move the endpoints to provisioned throughput (guaranteed, autoscaling capacity, billed as reserved DBUs rather than per-token).[2] Plus a flat-ish baseline for the App + serverless compute. None of this changes the per-document economics enough to move the ROI conclusion.
